Trong project front end này, ban đầu bạn đã xây dựng khung sườn cho tôi khá tốt, bây giờ sau khi tôi đã tạo db và các api cho phía backend ( trong thư mục OrderingFood_BE), tôi cần bàn hoàn thiện phần front end sao cho tương thích và phù hợp với phần backend này, ngoài ra sẽ phù hợp với tất cả các yêu cầu sau đây của tôi, tất nhiên trong quá trình hoàn thiện các yêu cầu này, nếu bạn cần thêm một số chức năng cho backend thì đừng ngần ngại và hãy code thẳng vào thư mục OrderingFood_BE,tuy nhiên, nếu được thì cũng hãy song song thực hiện nếu có thể thêm code ở Fe, tùy thuộc vào việc bạn thấy tiện nhất, bạn hãy tưởng tượng bạn là một nhà phát triển rất giỏi, sau đây là các yêu cầu:
Chú ý rằng các dữ liệu giờ đây sẽ bắt buộc sử dụng các api đã được code trong thư mục OrderingFood_DB
+) Khi mở app lên sẽ điều hướng trang đăng nhập đầu tiên, trang đăng nhập lúc này sẽ không chia ra là client hay seller nữa mà chỉ có 1 trang đăng nhập duy nhất, người dùng cần nhập email và mật khẩu để đăng nhập, front end sẽ get api từ backend để lấy người dùng có email và password giống như vậy, ở đây mỗi model users ( trong thư mục OrderingFood_BE) có quản lí roleID, nếu roleId là 1 (do tôi đã set roleId =1 là customer ) thì sẽ điều hướng đến giao diện cho customer, tức là khách hàng, nếu roleId =2 là seller thì sẽ điều hướng đến giao diện dành cho quản lý nhà hàng, tiếp theo về trang đăng kí, lúc này nếu chọn bạn là khách hàng thì sẽ post users mới với roleid=1, nếu chọn người bán hàng thì sẽ post users mới với roleid =2, đính kèm thêm các thông tin ( họ và tên,email, số điện thoại,… ( các trường bắt buộc để tạo mới 1 api user)), khi ấn nút đăng ký thì sẽ thực hiện chức năng ( nhớ rằng khi ấn đăng ký nếu roleid=1 thì sẽ đồng thời tạo cart tương ứng cho user này ), hãy hoàn thiện chức năng đăng nhập, đăng ký
